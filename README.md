# humanize-writing-skill【陈叶PPT设计开发】

> **去 AI 味，注入人味。**  
> 将 AI 生成的机械文本转化为有温度、有场景、有人类瑕疵的高质量内容。  
> *Inject soul into AI text. Remove robotic tone and add human touch to your writing.*

## 🧐 为什么需要这个技能？

你是否发现，AI 生成的文字总有股“味道”：

- ❌ **过度完美**：逻辑滴水不漏，却像机器人写的
- ❌ **空洞说教**：满篇“至关重要”、“赋能”、“闭环”
- ❌ **缺乏主体**：没有“我”的观点，只有“人们认为”
- ❌ **平衡句式**：“既...又...”、“一方面...另一方面”
- ❌ **陈词滥调**：“在...时代背景下”、“前所未有的机遇”

**这就是“AI 味”。** 它让文字失去灵魂，让读者感到疏离。

本技能基于 **Grammarly AI Humanizer 原理** 和 **中文写作实战案例**，提炼出 **「7 步手术法」**，专为去除 AI 味设计。

- 支持中文语境

### 安装

```bash
# 克隆本仓库
git clone https://github.com/Chenyeppt/humanize-writing-skill.git

# 将 SKILL.md 复制到你的 Open Claw 技能目录
cp SKILL.md ~/.openclaw/skills/humanize-writing/
```

### 使用

在你的对话中直接调用：

```
用 humanize-writing 技能优化这段文字：
"综上所述，人工智能技术的发展为各行各业带来了前所未有的机遇和挑战。"
```

**输出：**
```
别整那些"机遇与挑战"的虚词了。AI 就在那儿，要么它帮你干活，要么它帮你招人。
别想着"充分利用"，先想想怎么别让自家孩子输在起跑线上。
产业升级？那是后话。眼下是，你不用，你隔壁老王先用上了，他就比你快一步。
```

## 🧠 核心方法论：7 步手术法

| 步骤 | 操作 | 示例 (AI → 人类) |
|------|------|-----------------|
| **1️⃣ 词汇降级** | 用简单词替换复杂词 | `utilize` → `use`, `comprehensive` → `full` |
| **2️⃣ 删除强调词** | 去掉 "actually", "truly", "至关重要" | "actually feel" → "feel" |
| **3️⃣ 增加具体场景** | 用具体例子替换抽象概念 | "优化流程" → "把注册按钮放大两倍" |
| **4️⃣ 注入主观视角** | 加入 "我"、"我们" 等第一人称 | "应该" → "我觉得" |
| **5️⃣ 打破平衡句式** | 删除 "既...又..."，用短句代替长句 | "既提效又降本" → "效率上去了，成本？先别管" |
| **6️⃣ 增加停顿和反问** | 插入问句、感叹句、省略号 | "你想想看...", "但，那是昨天了" |
| **7️⃣ 承认局限性** | 表达不确定或错误 | "我也可能错了", "说实话，我也看不清" |

## 📚 实战案例

### 案例 1：商业文案

**❌ AI 原文：**
> 在数字化转型的浪潮中，企业必须认识到数据隐私保护的重要性。这不仅是合规的要求，更是建立用户信任的基石。

**✅ 人类重构：**
> 别跟我提什么"数字化转型"的大词。当用户发现他们的聊天记录被拿去卖了广告，信任瞬间就崩了。
> 这时候你再谈合规？晚了。**数据加密不是选择题，是你的用户在帮你守住最后一道门。**

### 案例 2：技术解释

**❌ AI 原文：**
> 这是把本地 8000 端口映射到远程服务器的 6000 端口。这样，手机访问服务器 6000 端口，就能连到本地电脑。

**✅ 人类重构：**
> 简单说，就像咱在烧烤店后厨通了条"暗道"。手机是客，服务器是前门，咱电脑是后厨。
> 现在前门不通，咱直接从外头挖条地道通到后厨去。只要对着暗号（端口），谁都能进来端菜。

### 案例 3：小说对话（《许昌双陈记》第 12 章）

**❌ AI 原文：**
> "技术不是为了炫技，是为了解决问题。如果'龙虾'能让各位少操心，让生意更好，那它就是有价值的。"

**✅ 人类重构：**
> "北京那是'高大上'的 AI，咱这是'泥腿子'的 AI。
> 北京那帮人看不上咱这土法子，但咱许昌的老板们认这个。
> 能帮他们省两斤羊肉钱，比啥都强。"

## 🎯 三个核心维度

### 1. 🌟 人设构建层 (Voice) - "谁在说话？"
- 植入烟火气：用具体生活场景切入
- 引入主观挣扎：允许出现"不确定性"
- 确立强立场：选边站，用强烈的判断力

### 2. 🧱 叙事节奏层 (Rhythm) - "怎样说话？"
- 用提问代替论断
- 制造阅读障碍的快感：长句后的突然短句
- 优化过渡词："说白了"、"关键点来了"

### 3. 🧠 思维模型层 (Substance) - "想什么？"
- 跨界联想：将 A 领域问题套用 B 领域模型
- 反直觉论述：提出违背常识的观察点

## 🛠️ 执行流程 (SOP)

### Phase 1: 诊断 (Diagnosis)
1. 扫描全文，标记"大词"（赋能、闭环、底层逻辑）
2. 标记"连接词"（因此、此外、综上所述）
3. 标记"无主语句"和"过度平衡句式"
4. 识别人设缺失维度（缺场景？缺情绪？缺观点？）

### Phase 2: 手术 (Surgery)
根据诊断结果，应用**7 步手术法**进行重构

### Phase 3: 验证 (Verification)
1. **朗读测试**：读出来，看哪里拗口
2. **图灵测试**：问自己，"这句话像真人在咖啡馆说的吗？"
